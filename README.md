# Feyn(wo)men’s Programming Lab

In our scientific endeavors, we frequently depend on the results of various algorithms.
But how often do we look under the hood?

- How is a list/set/dictionary implemented in Python?
- What does the ADAM optimizer actually do internally?
- Why is a random walk mostly sparse matrix multiplication, and how are sparse matrices implemented?

These are exactly the kinds of questions that deepen understanding and improve the soundness of scientific implementations.

This repository supports the **Feyn(wo)men’s Programming Lab at the LBI**, guided by:

> **“If you can build it, you understand it.”**

## The (Extended) Feynman Method

We use a four-step approach to reach deep mastery:

1. **Study It**: Delve into the core principles and logic of the concept.
2. **Teach It**: Explain the concept in a straightforward manner to someone else.
3. **Spot Gaps**: Identify areas where understanding falls short.
4. **Build It**: Implement the concept from scratch to demonstrate mastery.

## What We’ll Cover

Using Rust (or any language of your choice), we explore topics such as:

- Basic algorithms and data structures
- Linear algebra
- Graph algorithms
- Optimization
- and more

## Weekly Workflow

Each session follows the same structure:

1. **Presentation**: A rotating speaker presents an algorithm of their choice.
2. **Implementation from Scratch**: We implement the algorithm using only our own code:
   - no external libraries
   - and, most importantly, no LLMs
